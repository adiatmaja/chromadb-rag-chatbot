# Data Directory

This directory contains sample data files used to populate the ChromaDB vector collections.

> **Note:** The files in this directory are **demo/sample data** for illustration purposes.
> In a production deployment, replace these with your actual product catalog, stock data,
> and intent definitions.

## Files

| File | Description |
|------|-------------|
| `product_data.csv` | Product catalog with SKUs, official names, colloquial aliases, and pack sizes |
| `stock_data.csv` | Inventory stock levels per SKU and warehouse location |
| `faq_data.csv` | Sample FAQ entries for CSV-based FAQ indexing (no ClickHouse required) |
| `intent.txt` | Intent definitions and training examples for intent classification |
| `intent_data.csv` | Parsed intent data (generated by `scripts/indexing/parse_intent_data.py`) |

## product_data.csv Schema

```csv
sku,official_name,colloquial_names,pack_size_desc,embedding_text
```

- `sku` — Unique product identifier
- `official_name` — Full official product name
- `colloquial_names` — Semicolon-separated list of common/colloquial names customers use
- `pack_size_desc` — Description of packaging unit
- `embedding_text` — Pre-formatted text used for semantic embedding generation

## stock_data.csv Schema

```csv
sku,product_name,stock_quantity,unit,warehouse_location,last_updated
```

## intent.txt Format

```
** intent_name: Description of the intent

Examples:
Customer Message: 'example phrase', intent: 'intent_name'
```

Run `python scripts/indexing/parse_intent_data.py` to generate `intent_data.csv` from `intent.txt`.

## faq_data.csv Schema

```csv
id,question,answer,language,created_at,updated_at
```

Use `scripts/indexing/index_faq_csv.py` to index FAQs from this file without needing ClickHouse.
For production (ClickHouse source) use `scripts/indexing/index_faq.py` instead.
